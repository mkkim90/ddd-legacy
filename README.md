# 키친포스

## 퀵 스타트

```sh
cd docker
docker compose -p kitchenpos up -d
```

## 요구 사항
- 상품 요구 사항
  - 상품을 생성한다
    - [] 가격은 필수값이며, 0보다 커야한다.
    - [] 상품명은 필수값이며, 비속어가 포함되어서는 안된다.
  - 상품을 수정한다
    - [] 가격은 필수값이며, 0보다 커야한다. 
    - [] 수정하고자 하는 상품이 존재해야한다.
    - [] 수정한 가격이 해당 상품이 포함된 메뉴에 가격에 적용이 되어야한다. 합산 가격이 일치하지 않으면 해당 상품을 비노출한다.
  - 상품을 조회한다
    - [] 해당 상품을 전체 조회한다.
- 메뉴 요구 사항
  - 메뉴를 생성한다
    - [] 가격은 필수값이며, 0보다 커야한다. 
    - [] 생성하고자 하는 메뉴의 그룹이 존재해야한다.
    - [] 메뉴상품은 필수 값이며, 하나 이상이어야한다.
    - [] 메뉴상품은 상품테이블에 존재해야한다.
    - [] 메뉴상품의 갯수는 0보다 커야한다.
    - [] 메뉴상품 가격의 합산 가격이 일치해야한다. 
    - [] 메뉴명은 필수값이며 비속어를 포함하지 않는다.
  - 메뉴를 수정한다
    - [] 가격은 필수값이며, 0보다 커야한다. 
    - [] 수정하고자하는 메뉴가 존재해야한다.
    - [] 수정하고자하는 가격이 메뉴상품의 합산 가격이 일치해야한다.
  - 메뉴를 노출한다
    - [] 노출하고자 하는 메뉴가 존재해야한다.
    - [] 노출하고자하는 메뉴가격이 메뉴상품의 합산 가격과 일치해야한다.
  - 메뉴를 비노출한다.
    - [] 비노출하고자 하는 메뉴가 존재해야한다.
  - 메뉴를 조회한다. 
    - [] 전체 메뉴를 조회한다. 
- 메뉴 그룹 요구 사항
  - 메뉴 그룹을 생성한다
    - [] 메뉴 그룹명은 필수값이다. 
  - 메뉴 그룹을 조회한다.
    - [] 전체 메뉴그룹을 조회한다. 
- 주문 요구 사항
  - 주문을 생성한다
    - [] 주문타입은 필수 값이어야 한다. 
    - [] 메뉴 주문은 필수 값이어야 한다. 하나 이상이어야한다.
    - [] 메뉴 주문의 메뉴는 존재해야한다. 
    - [] 메뉴 주문 수량은 먹고 가는 주문이 아닌 경우 0보다 커야한다.
    - [] 메뉴 주문의 메뉴는 노출 메뉴여야 한다. 
    - [] 메뉴 주문의 가격은 메뉴 가격과 동일해야한다. 
    - [] 주문 생성시 주문 상태값은 대기여야한다. 
    - [] 주문 타입이 배달인 경우 배달 주소는 필수 값이어야한다.
    - [] 주문 타입이 먹고가기인 경우 주문 테이블은 존재해야하고 비어있어야한다. 
  - 주문을 접수한다
    - [] 주문이 존재해야한다.
    - [] 현재 주문상태가 대기여야한다.
    - [] 주문타입이 배달인 경우 배달 접수한다.
  - 주문을 준비한다
    - [] 주문이 존재해야한다.
    - [] 현재 주문상태가 접수여야 한다. 
  - 주문을 배달한다.
    - [] 주문이 존재해야한다.
    - [] 주문 타입이 배달이어야한다.
    - [] 현재 주문상태가 준비여야 한다. 
  - 주문을 배달 완료한다.
    - [] 주문이 존재해야한다.
    - [] 주문 타입이 배달이어야한다.
    - [] 현재 주문상태가 배달중이여야 한다.
  - 주문을 최종 완료한다.
    - [] 주문이 존재해야한다.
    - [] 주문 타입이 배달인 경우 현재 주문상태가 배달 완료여야 한다.
    - [] 주문 타입이 포장 또는 먹고가기인 경우 현재 주문상태가 준비여야 한다. 
    - [] 주문 타입이 먹고 가기인 경우 주문 테이블을 비워둔다 
  - 주문을 조회한다
    - [] 전체 조회한다.
